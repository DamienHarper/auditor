# Upgrade from 4.x to 5.0

This guide covers the changes needed when upgrading from auditor 4.x to 5.0.

## Overview

auditor 5.0 completes the extraction of the Doctrine ORM provider into the standalone
[`damienharper/auditor-doctrine-provider`](https://github.com/DamienHarper/auditor-doctrine-provider)
package. The `DH\Auditor\Provider\Doctrine` namespace has been **removed** from the core library.

The class FQCNs are identical in the external package â€” no code changes are required for most users.

## Migration Steps

### 1. Install the Doctrine provider package (if not already installed)

```bash
composer require damienharper/auditor-doctrine-provider:^2.0
```

### 2. Update auditor itself

```bash
composer require damienharper/auditor:^5.0
```

### 3. Update `NeedsConversionToAuditableType` imports (if applicable)

The interface has moved from the core package to the doctrine-provider package.

```php
// Before (4.x)
use DH\Auditor\Transaction\NeedsConversionToAuditableType;

// After (5.0)
use DH\Auditor\Provider\Doctrine\Transaction\NeedsConversionToAuditableType;
```

## Breaking Changes

### Removed: `DH\Auditor\Provider\Doctrine\*` from core

The entire `src/Provider/Doctrine/` namespace has been removed from `damienharper/auditor`.
It is now exclusively provided by `damienharper/auditor-doctrine-provider` under the same namespace.

### Removed: `DH\Auditor\Transaction\NeedsConversionToAuditableType`

This interface (introduced in 4.1, PR #309) required `doctrine/dbal` as a core dependency.
It has moved to `DH\Auditor\Provider\Doctrine\Transaction\NeedsConversionToAuditableType`
in `auditor-doctrine-provider`.

### Removed: Doctrine packages from core `require-dev`

`doctrine/dbal`, `doctrine/orm`, `doctrine/data-fixtures`, and `gedmo/doctrine-extensions`
are no longer installed when running `composer install` on `damienharper/auditor` alone.

## Requirements Changes

| Requirement   | 4.x     | 5.0     |
|---------------|---------|---------|
| PHP           | >= 8.4  | >= 8.4  |
| Symfony       | >= 8.0  | >= 8.0  |
| Doctrine DBAL | >= 4.0  | via `auditor-doctrine-provider` |
| Doctrine ORM  | >= 3.2  | via `auditor-doctrine-provider` |

## Upgrading from 5.x intermediate releases

### `Entry::getDiffs()` return type change

**Who is affected:** Any code that calls `$entry->getDiffs()` and treats the result as an
`array` (e.g. `foreach`, `array_key_exists`, passing to array functions).

**Before (array):**

```php
$diffs = $entry->getDiffs(); // array

foreach ($diffs as $field => $change) {
    $old = $change['old'] ?? null;
    $new = $change['new'];
}

// REMOVE entry:
$class = $diffs['class'];

// ASSOCIATE entry:
$sourceClass = $diffs['source']['class'];
```

**After (`AuditDiffCollection`):**

```php
use DH\Auditor\Model\DiffKind;

$diffs = $entry->getDiffs(); // AuditDiffCollection

// INSERT / UPDATE â€” still iterable:
foreach ($diffs as $field => $diff) {   // $diff is AuditDiff
    $old = $diff->old;                  // null when hasOld is false (INSERT)
    $new = $diff->new;
    $hasOld = $diff->hasOld;            // false for INSERT, true for UPDATE
}

// REMOVE entry:
$snapshot = $diffs->requireEntitySnapshot();
$class = $snapshot->class;

// ASSOCIATE / DISSOCIATE entry:
$rel = $diffs->requireRelationDescriptor();
$sourceClass = $rel->source->class;
```

Branch on `$diffs->kind` to dispatch between the three structural variants:

```php
match ($diffs->kind) {
    DiffKind::FieldChanges => ...,
    DiffKind::EntityRemoval => $diffs->requireEntitySnapshot(),
    DiffKind::Associate,
    DiffKind::Dissociate   => $diffs->requireRelationDescriptor(),
};
```

See the [Reading Audit Diffs](../querying/audit-diff.md) guide for the full API.

**auditor-bundle:** The bundle's Twig templates (`entry.html.twig`) access `entry.diffs`
which maps to `getDiffs()`. These templates must be updated to work with `AuditDiffCollection`.
A compatible version of `auditor-bundle` that handles the new types will be released alongside
this version of auditor.
